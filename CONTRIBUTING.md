# 参与贡献指南

感谢你希望为暗夜图鉴相机增添新的居民、夜班或交互。这个项目的重点不是把现实人物变成被评价的对象，而是共同扩张一个可被探索的虚构世界。

## 开始之前

1. 先查看 [产品与体验系统](docs/product-system.md) 和 [内容安全边界](docs/content-safety.md)。
2. 对新玩法、居民、主题周或大型体验改动，请先创建 Feature Request，说明目标、用户价值和边界。
3. UI、文本和数据变更都应在 390 × 844 的移动视口中验证。

## 本地开发

```bash
cd app
pnpm install
pnpm dev
pnpm build
```

## 内容规范

- 图鉴角色必须是原创的虚构存在；不要复刻特定影视、动画、游戏或漫画角色。
- 不得根据上传照片推断现实人格、身份、种族、健康、吸引力或情绪。
- 惊悚感可来自异化、职业、空间和不可靠规则；避免对真实用户的羞辱、贬损或血腥伤害描写。
- 新剧情的失败/失落结局必须明确是虚构角色的世界线后果，不能暗示对真实用户的预测。

## 代码与数据约定

- 优先使用数据驱动结构扩展图鉴和剧情，避免将角色专属逻辑散落在 UI 渲染层。
- 所有可见文案保持简体中文；关键安全声明应在对应入口附近可见。
- 新增图片需使用清晰、可追溯的命名，例如 `mbti-mirror-gate-peddler.png`。
- 不提交 `.env`、构建产物、运行日志、私密账户文件或未经确认的人像素材。
